# 规划:四视角评审落地 —— 多会话并行 + 不冲突 + 合得回来

> 本文是**并行多会话的唯一协调文档**。每个会话开工前先读这份,认领一个会话编号,
> **只碰自己 territory 的文件**,持续打单关注点 PR 合进 `main`(随合随集成)。
> 基线:origin/main `3730ac5`(四视角评审 6 PR #53–#58 已合并)。

---

## 0. 为什么能并行(核心原则)

1. **Territory 隔离 = 不冲突的根**。冲突只来自两个会话改同一个文件。下面给每个会话划**专属文件集** + **明确禁碰清单**;热点文件(`agent.go` / `controller.go` / `terminal.go` / `config.go`)**各自只归一个会话**。
2. **随合随集成**,不搞最后大爆炸合并。每个会话像之前一样:独立 detached worktree(`git worktree add --detach /tmp/lumen-<s> origin/main`)、TDD 红→绿、`make check`+`-race` 绿、单关注点 squash PR、趁网络窗口合。territory 不重叠 → squash 合并天然无冲突。
3. **eval 是共享脊柱**。质量/性能/裁剪都对着 `lumen eval` 量化。会话 1 拥有它;别的会话**只新增任务目录**(命名空间隔离),不碰 `internal/eval/*.go`。
4. **改行为的、要 measure-before-cut 的,门槛是 eval**。没有真实基线前,不盲删、不翻默认。
5. **诚实**:每个 PR 描述写清"验证了什么/没验证什么";改不了的(余额/无 key)如实标 resource-gated。

## 2. 四个并行会话

### 🟦 S1 — Eval 与度量(共享脊柱)
**Mission**:把"质量靠嘴说"变成可量化的 pass-rate,并扩任务集 + 进 CI。
**Territory(可改)**:`internal/eval/**`、`evals/tasks/**`(新目录)、`Makefile`(eval 段)、`.github/workflows/`(新增 eval 相关 job)、`docs/eval-*.md`。
**禁碰**:`agent.go`/`controller.go`/`terminal.go`/`config.go`/`guard`/任何业务包。
**交付**:
- PR-1:任务集 3 → ~25(多语言 Go/Py/JS;类别:bugfix / 实现新函数 / 重构 / 边界 / 失败测试修复;难度分层)。命名 `NN-<slug>`。每个**初始必失败**(no-op 得 0)。
- PR-2:`lumen eval` 增 `--json`(输出机器可读结果)+ `--model`/`--profile` 透传,便于 full-vs-core、DeepSeek-vs-local 对比。
- PR-3:CI 加 `eval-selfcheck` job(只跑 `internal/eval` 的确定性测试 + `lumen eval --list` 校验任务集格式;**不**跑活模型)。
- PR-4(**resource-gated**):用真实模型跑一次,把基线 pass-rate / median-steps / cost 写进 `docs/eval-baseline.md`。需 DeepSeek 充值 **或** 本地 LM Studio Qwen。
**门槛**:PR-1~3 现在就能做;PR-4 等模型。
**协作**:别的会话要加 eval 任务 → 用自己的命名空间目录(S3=`sec-*`),不碰 `eval.go`。

### 🟩 S2 — 系统 / 性能 / 路由(Huang 线;独占 agent 循环)
**Mission**:把"造了没接"的引擎接上 + 让性能可测 + 本地推理一等公民。
**Territory(可改)**:`internal/orchestrator/**`、`internal/modelpool/**`、`internal/perf/**`(新)、`internal/config/model_presets.go`、`internal/config/config.go`、`internal/agent/agent.go`、`cmd/lumen/terminal.go`、`internal/control/controller.go`、`internal/control/{config_apply,tools_profile}.go`。
**禁碰**:`internal/eval`、`internal/guard`、待删包、README/wizard。
**交付**:
- PR-1:**perf HUD** —— 新 `internal/perf`,经 event 发 TTFT / tokens-per-sec / turn 墙钟 / cost;sink 渲染一行 footer/HUD。(token 估算顺手统一成真实 API usage。)
- PR-2:**接 orchestrator** —— 并行子 agent(task 工具背后真并行),验证用 `-race`。
- PR-3:**接 modelpool** —— 成本/延迟感知路由 + 真实 failover(别再重放整个 prompt)。
- PR-4:**本地推理一等公民** —— `model_presets.go` 加 LM Studio / Ollama / vLLM 预设(OpenAI 兼容 base_url),`lumen doctor` 探测本地 endpoint(doctor 若属 S4,则只在 presets 内加,探测留 S4);本地↔云路由开关。
**门槛**:PR-1~3 现在能做;PR-4 的"活验证"需本地模型起。**翻 `[tools] profile` 默认到 core 不在这做** —— 那是 eval-gated,留集成期由 eval 数据定。
**协作**:**S3 的 audit 钩子**(`agent.go` 一行)由本会话代接 —— 等 S3 给出 `audit.Record(...)` 签名。

### 🟧 S3 — 安全 / 信任(Amodei 线;不碰 agent 循环)
**Mission**:把 denylist 升级到有沙箱 + 审计可回答"agent 为什么这么干" + 注入隔离 + 写威胁模型。
**Territory(可改)**:`internal/guard/**`、`internal/sandbox/**`(新)、`internal/audit/**`(新/现有)、`internal/tool/builtin/*`(仅工具输出包裹层 + sandbox 接 bash)、`docs/threat-model.md`(新)、`evals/tasks/sec-*`(注入/危险命令语料,命名空间隔离)。
**禁碰**:`agent.go`/`controller.go`/`terminal.go`/`internal/eval/*.go`/README。
**交付**:
- PR-1:`docs/threat-model.md` —— 资产、攻击面(它跑 rm/curl/写文件)、信任边界、已缓解 vs 未缓解、为什么 denylist≠sandbox。纯文档,零冲突。
- PR-2:**guard 属性/对抗测试** —— 把 guard 从样例测试升级到 property-based(随机混淆/编码/空格/Unicode),诚实声明覆盖边界。
- PR-3:**沙箱脚手架** —— 新 `internal/sandbox`(可用 `sandbox-exec`(mac)/bwrap/容器,抽象 Runner),bash 工具可选在沙箱内跑(配置开关,默认 off 以不改行为)。
- PR-4:**审计轨迹** —— `internal/audit` store(落盘 JSONL,记录每个工具调用的 why/args/结果),提供 `audit.Record` 签名给 S2 接一行。`audit_query` 工具读它。
- PR-5:**注入隔离** —— 在**工具输出层**(不在 agent.go)给 web_fetch / file-read / 工具输出包一层 "untrusted content" 标记 + web_fetch 加 SSRF 过滤。
**门槛**:全部现在能做(纯本地,无需模型)。
**协作**:PR-4 的 `agent.go` 钩子交给 S2。

### 🟪 S4 — 产品聚焦 / 瘦身(Altman 线;eval-gated 的删除)

## 3. 集成 / 合并方案

- **随合随集成**:四会话各自 squash 进 `main`,territory 不重叠 → 无冲突。任何会话合完,别的会话**下次开 PR 前 `git fetch` + 基于最新 main**(踩过的坑:基于过期 main 建分支)。
- **唯一跨会话接线**:S3 的 `audit.Record` → S2 在 `agent.go` 接一行。S3 先合 audit 包,S2 fetch 后接。若两会话节奏错开,留到集成期由我接。
- **集成收尾(我或一个收尾会话)**:
  1. 全部合完后,在最新 main 上 `make check` + `-race ./...` + `lumen eval`(full vs core、DeepSeek vs local 各一次)确认 composition 绿、pass-rate 不退。
  2. 据 eval 数据定**是否翻 `[tools] profile` 默认到 core**(S2 留的 eval-gated 决定)。
  3. `make facts` 刷新 README 计数;更新 auto-memory `lumen-project.md`。
  4. 写一份 `docs/交接-v7-评审落地合并.md` 汇总。
