# 规划:四视角评审落地 —— 多会话并行 + 不冲突 + 合得回来

> 本文是**并行多会话的唯一协调文档**。每个会话开工前先读这份,认领一个会话编号,
> **只碰自己 territory 的文件**,持续打单关注点 PR 合进 `main`(随合随集成)。
> 基线:origin/main `3730ac5`(四视角评审 6 PR #53–#58 已合并)。

---

## 0. 为什么能并行(核心原则)

1. **Territory 隔离 = 不冲突的根**。冲突只来自两个会话改同一个文件。下面给每个会话划**专属文件集** + **明确禁碰清单**;热点文件(`agent.go` / `controller.go` / `terminal.go` / `config.go`)**各自只归一个会话**。
2. **随合随集成**,不搞最后大爆炸合并。每个会话像之前一样:独立 detached worktree(`git worktree add --detach /tmp/lumen-<s> origin/main`)、TDD 红→绿、`make check`+`-race` 绿、单关注点 squash PR、趁网络窗口合。territory 不重叠 → squash 合并天然无冲突。
3. **eval 是共享脊柱**。质量/性能/裁剪都对着 `lumen eval` 量化。会话 1 拥有它;别的会话**只新增任务目录**(命名空间隔离),不碰 `internal/eval/*.go`。
4. **改行为的、要 measure-before-cut 的,门槛是 eval**。没有真实基线前,不盲删、不翻默认。
5. **诚实**:每个 PR 描述写清"验证了什么/没验证什么";改不了的如实标 gated。

## 0.5 本地模型优先(本轮主引擎,DeepSeek 太烧钱)

**所有测试 / eval / dogfood / 删包回归,默认对着本地模型跑(免费、无限)**,DeepSeek 仅作偶尔参照。
本地模型即将就绪(下载中):

| 模型 | 大小 | 工具调用(决定能否驱动 agent 循环) | 用途 |
|---|---|---|---|
| **qwen/qwen3.6-27b** | ~16GB | ✅ 强(Qwen 系以 tool-use 著称) | **eval 基线主力 + dogfood 主力** |
| Gemma 4 12B Coder(Fable/Composer 社区微调) | ~7.4GB | ⚠️ **待实测**(Gemma 历史 function-calling 弱/不稳) | 对照;能 tool-call 才纳入 eval |
| Gemma 3 / 更小 | — | ⚠️ 待实测 | 对照 / 快速冒烟 |

运行方式:**LM Studio 的 OpenAI 兼容服务**(`http://localhost:1234/v1`,GUI 里 Start Server)或已下载的 `llama.cpp-mac-arm64-metal` server。Lumen 接法已验证可行:`[[providers]] kind="openai" base_url="http://localhost:1234/v1" api_key="lm-studio"`(key 随便填,空 key 也行)。

**地基铁律(本轮头号前置)**:
1. **先做"本地接入 + 能力验证"(= S2 的第一个 PR),它 gate 住 S1 基线 / S4 删包回归 / 全员 dogfood。**
2. **工具调用能力必须每模型实测**:跑一个"读文件→改一行→verify 过"的最小任务,确认该模型真能发 OpenAI `tool_call`。**Qwen 预期过;Gemma 可能发不出**(则只会"嘴上说改"却调不动 `edit_file`)→ 产出一张**能力矩阵**(模型 × 能否驱动 agent);eval 只跑"能驱动"的模型,Gemma 不行就降级为对照或标 gated。
3. **新瓶颈是延迟不是钱**:本地 27B 在 Mac/Metal 上比 API 慢 → perf HUD(S2)和 eval 的 latency 列变重要;cost 列对本地恒 ~$0。

## 2. 四个并行会话

### 🟦 S1 — Eval 与度量(共享脊柱)
**Mission**:把"质量靠嘴说"变成可量化的 pass-rate,并扩任务集 + 进 CI。
**Territory(可改)**:`internal/eval/**`、`evals/tasks/**`(新目录)、`Makefile`(eval 段)、`.github/workflows/`(新增 eval 相关 job)、`docs/eval-*.md`。
**禁碰**:`agent.go`/`controller.go`/`terminal.go`/`config.go`/`guard`/任何业务包。
**交付**:
- PR-1:任务集 3 → ~25(多语言 Go/Py/JS;类别:bugfix / 实现新函数 / 重构 / 边界 / 失败测试修复;难度分层)。命名 `NN-<slug>`。每个**初始必失败**(no-op 得 0)。
- PR-2:`lumen eval` 增 `--json`(机器可读)+ `--model`/`--profile`/`--base-url` 透传 + **`--repeat N`**(本地模型非确定性大,跑多次取均值)+ 报告里 **latency 列**(本地的真瓶颈);让"**本地模型 A vs B vs Qwen vs(可选)DeepSeek 参照**"和 full-vs-core 都能一键比。
- PR-3:CI 加 `eval-selfcheck` job(只跑 `internal/eval` 确定性测试 + `lumen eval --list` 校验格式;**不**跑活模型)。
- PR-4(**本地模型-gated,模型就绪即可,无需花钱**):对着**本地 Qwen3.6-27b** 跑基线,产出 `docs/eval-baseline.md`:每个"能驱动 agent"的本地模型一行(pass-rate / median-steps / median-latency / cost≈$0),并标注 Gemma 是否能 tool-call(来自 S2 能力矩阵)。
**门槛**:PR-1~3 现在就能做;PR-4 等 **S2-PR1(本地接入+能力矩阵)落地 + 模型下载完**(都是临门一脚,不烧钱)。
**协作**:别的会话加 eval 任务 → 用自己命名空间(S3=`sec-*`),不碰 `eval.go`。基线主力模型取自 S2 的能力矩阵(只跑"能驱动"的)。

### 🟩 S2 — 本地推理 + 系统 / 性能 / 路由(Huang 线;独占 agent 循环)
**Mission**:**先让本地模型一等公民可用(地基)**,再把"造了没接"的引擎接上 + 让性能可测。
**Territory(可改)**:`internal/orchestrator/**`、`internal/modelpool/**`、`internal/perf/**`(新)、`internal/config/model_presets.go`、`internal/config/config.go`、`internal/agent/agent.go`、`cmd/lumen/terminal.go`、`internal/control/controller.go`、`internal/control/{config_apply,tools_profile}.go`、`docs/local-models.md`(新)。
**禁碰**:`internal/eval`、`internal/guard`、待删包、README/wizard。
**交付(PR-1 是全局头号前置,先做、先合)**:
- **PR-1:本地推理一等公民 + 能力矩阵(地基,gate 住 S1-PR4 / S4 删包 / 全员 dogfood)**
  - `model_presets.go` 加 **LM Studio / Ollama / vLLM** 预设(OpenAI 兼容 `base_url`,空/假 key 可用)。
  - 一个**能力探测**(脚本或 `lumen doctor --local`):对每个本地模型跑"读文件→改一行→verify 过"的最小任务,记录**能否发 OpenAI `tool_call`** → 写 `docs/local-models.md` 能力矩阵(模型 × 驱动 agent ✓/✗ × 大致 tokens/sec)。
  - 若某模型(可能是 Gemma)发不出 tool_call:记录为"对话/补全可用、agent 不可用",并评估是否值得加一个**轻量 tool-call 适配层**(把模型的文本意图解析成工具调用)——**评估结论写进矩阵,不强行实现**。

- PR-2:**perf HUD** —— 新 `internal/perf`,经 event 发 TTFT / tokens-per-sec / turn 墙钟 / latency;sink 渲染一行(本地延迟是主指标)。token 估算顺手统一成真实 usage。

## 3. 集成 / 合并方案

- **随合随集成**:四会话各自 squash 进 `main`,territory 不重叠 → 无冲突。任何会话合完,别的会话**下次开 PR 前 `git fetch` + 基于最新 main**(踩过的坑:基于过期 main 建分支)。
- **唯一跨会话接线**:S3 的 `audit.Record` → S2 在 `agent.go` 接一行。S3 先合 audit 包,S2 fetch 后接。若两会话节奏错开,留到集成期由我接。
- **集成收尾(我或一个收尾会话)**:
  1. 全部合完后,在最新 main 上 `make check` + `-race ./...` + `lumen eval`(**对本地 Qwen**:full vs core 各一次;Gemma 若能驱动则也跑)确认 composition 绿、pass-rate 不退。
  2. 据 eval 数据定**是否翻 `[tools] profile` 默认到 core**(S2 留的 eval-gated 决定)。
  3. `make facts` 刷新 README 计数;更新 auto-memory `lumen-project.md`。
  4. 写一份 `docs/交接-v7-评审落地合并.md` 汇总。
