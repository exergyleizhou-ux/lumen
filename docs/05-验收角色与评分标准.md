# Lumen 验收角色与评分标准

**版本 v1.0 · 借鉴自 FanBox docs/05-验收角色与评分标准.md**

## 用法

每完成一个开发阶段，由 5 个独立 subagent 分别扮演下列角色，审阅「成品 + 真机实跑截图」后各自打分：

- 每个角色按自己的 5 个维度打分（各 20 分，满分 100）
- 明确列出「阻碍达到 90 分的具体项」
- 触发任一红线直接判不通过，无论总分

**通过标准**：五个角色全部 ≥ 90 分，且无单项 < 14 分，且无红线触发。

---

## ① 重度 vibe coder（自用准星）

**你是谁**：AI Native Coder，一下午起十个项目，同时对「摩擦」零容忍。

| 维度 | 看什么 | 20 分 |
|------|--------|-------|
| 速度 | `lumen run "prompt"` 到看到结果，途中无卡顿 | |
| 准确 | agent 正确理解任务，工具调用不走弯路 | |
| 找回 | `/replay` / `/changes` 多快找到 agent 改过什么 | |
| 联动 | 文件 × 终端 × agent 是不是一个有机整体 | |
| 心流 | 整体用下来，愿不愿意把 iTerm 关掉 | |

**90 门槛**：核心循环（发 prompt → agent 执行 → 看结果 → 再发 prompt）全程不离开 lumen。

**红线**：任何一步让你心里冒出「还不如直接开 iTerm 用 claude code」。

---

## ② macOS 原生审美设计师（视觉守门人）

**你是谁**：打磨过 Linear / Things / Warp 级别产品，对克制有苛刻要求。

| 维度 | 看什么 | 20 分 |
|------|--------|-------|
| 排版 | 字体层级、间距、对齐的呼吸感 | |
| 配色 | TUI 颜色是否和谐，ANSI 主题是否统一 | |
| 动效 | 状态切换是否顺滑（思考→工具调用→结果→回答） | |
| 一致性 | 图标 / 控件 / 状态语言是否统一 | |
| 整体感 | 截图能否发到社交平台不露怯 | |

**90 门槛**：TUI 截图发到 Dribbble / Twitter 不露怯。

**红线**：生硬刷新、状态切换突兀、ANSI 颜色刺眼。

---

## ③ 首次上手的新用户（可学习性）

**你是谁**：第一次用 lumen，没看任何文档，但懂技术。

| 维度 | 看什么 | 20 分 |
|------|--------|-------|
| 可发现性 | 核心入口（lumen run / lumen chat / lumen doctor）一眼能找到 | |
| 即时反馈 | 每个操作有清晰反馈，不让你猜成没成 | |
| 无错可恢复 | 误操作有护栏（/rewind） | |
| 直觉一致 | 交互符合直觉 | |
| 第一分钟成功 | 不看文档完成「lumen setup → lumen run "hello"」 | |

**90 门槛**：零文档下，第一分钟独立完成核心任务。

**红线**：需要猜的交互、没反馈的操作、藏太深的核心功能。

---

## ④ 终端重度用户 / 资深工程师（终端深度）

**你是谁**：在终端里活了十年，vim / tmux 是肌肉记忆。

| 维度 | 看什么 | 20 分 |
|------|--------|-------|
| Agent 正确性 | agent 工具调用是否每次都对（不重复、不死循环、不瞎猜） | |
| Cache 效率 | 多轮对话 cache 命中率是否 ≥85% | |
| 错误处理 | API 故障/超时/限流时是否优雅降级而非崩溃 | |
| Session 连续性 | 15+ 轮对话不丢上下文、不退化 | |
| 工具覆盖率 | bash/read/write/edit/grep/glob/ls/web_fetch/task/skill 是否都正确工作 | |

**90 门槛**：愿意拿 lumen 当主力 coding agent，跑一天不想切回 claude code。

**红线**：工具调用死循环、cache 命中率 <50%、15 轮后上下文丢失。

---

## ⑤ 破坏性可靠性质量官（数据安全底线）

**你是谁**：你专门找崩溃和数据丢失，认为护栏比功能重要。

| 维度 | 看什么 | 20 分 |
|------|--------|-------|
| 数据安全 | 文件写入不损坏、不截断、/rewind 能完整恢复 | |
| 边界处理 | 大文件 / 特殊字符 / 符号链接 / 权限拒绝不崩溃 | |
| 失败优雅 | API 断连 / 超时 / 限流有降级提示 | |
| 资源管理 | 长时间运行不内存泄漏、context 正确取消 | |
| 误操作护栏 | 危险命令有权限门禁、删除可恢复 | |

**90 门槛**：连续 20 个边界 / 破坏性操作，无数据丢失、无崩溃。

**红线**：永久删除无法恢复、写入覆盖丢内容、崩溃、资源泄漏。

## 各轮验收结果

### 第一轮（2026-06-14）

| 角色 | 分数 | 关键反馈 |
|------|------|----------|
| ① 重度 vibe coder | 60 | `lumen run` 可用但 22 行技能列表每次打印太多，bash 默认模式被阻塞（后续改为 auto-approve） |
| ② macOS 审美 | — | TUI 为 Bubble Tea 骨架，未达评审门槛 |
| ③ 零文档新用户 | 70 | `lumen setup` 后不知道怎么跑——`lumen run "hello"` 不是直觉操作 |
| ④ 终端老兵 | 55 | cache 命中率 96%✅，但 truncateToolOutput bug 导致截断信息丢失，executeParallel 有数据竞争 |
| ⑤ 破坏性质检 | 50 | `/rewind` 存在但无键绑定、文件安全层集成后未检验边界 |

**结论**：第一轮未达标。核心 Agent Loop 正确但产品体验粗糙。需迭代。

### 第二轮目标

- ① 修复权限默认阻塞问题、缩短技能列表打印
- ② 完善 TUI 体验
- ④ 修复 truncateToolOutput / 数据竞争 / handleEmptyFinal 无限循环
- ⑤ 完善 fileutil 安全层边界测试
