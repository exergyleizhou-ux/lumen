## 2. ⚠️ 本地模型硬件天花板(实测,本轮最重要的现实发现)

用户要从 DeepSeek("太烧钱")转本地模型(LM Studio,OpenAI 兼容 `:1234`)。实测(用户机器:
**MacBook Air, 24GB**):

- **Qwen3.6-27B(17GB)→ 内核 panic ×2**(watchdog timeout:17GB 模型饿死 24GB 系统)。已删。**别再碰 ≥~13GB 模型。**
- **Gemma4-12B 创意微调(fable/composer,7.4GB)→ 装得下、能调一次工具,但撑不住 agent 循环**:第一次 tool_call 后空回复→胡言,从不 `edit_file`;首字延迟 ~41s。是写作/角色微调,非 agent 微调。
- **结论**:该机器现有本地模型都驱动不了 Lumen 的多步 agent 循环。整条 Lumen↔LM Studio 链路本身**是通的**(`kind="openai" base_url=:1234` + `[tools] profile="core"` 让 prompt 塞得下小上下文)。
- **推荐**:`Qwen2.5-Coder-7B-Instruct`(Q4 ≈4.5GB,装得下、快、Qwen 工具调用强)。详见 auto-memory `user-model-setup`。

LM Studio 操作:`~/.lmstudio/bin/lms server start` + `lms load <m> -c <ctx> -y`。
**坑**:server 守护进程**不跨 sandbox Bash 调用存活** → start+load+`lumen run` 必须同一次调用。
