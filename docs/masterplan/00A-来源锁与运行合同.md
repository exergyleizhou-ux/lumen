## 1. 来源锁

| 工件 | 路径 | 规则 |
|------|------|------|
| SOURCE_LOCK | 仓库根 `SOURCE_LOCK.json` | 记录 monorepo HEAD、agent pin、生成时间 |
| IMPORT_LEDGER | `IMPORT_LEDGER.md` | 迁移目录与策略（历史 + 当前） |
| UPSTREAM | `agent/UPSTREAM.md` | Grok pin 政策 |

**漂移：** source tuple / 发布二进制变更 → 旧 readiness evidence 标 `INVALIDATED`，完整重跑。

**Secret gate：** 发布前 `gitleaks`（工具缺失则 BLOCKED，不静默跳过）。  
生成/刷新：`./scripts/source-lock.sh`

## 2. Run / Event / Effect（单一权威）

权威落在 **Grok session 目录 / JSONL**，不新建第二 DB。

| ID | 合同 | 最低验收 |
|----|------|----------|
| R1 | created→running→terminal；终态不可逆 | 合同测试 / 文档对照 session 状态机 |
| R2 | 单 writer | 禁止平行 run DB |
| R3 | seq 单调；tool_call_id 幂等 | 重复 effect 不双执行 |
| R4 | after_seq 有界补拉 | 断线只补缺口 |
| R5 | hard cancel 贯穿子进程树 | cancel 后无泄漏进程 |
| R6 | kill -9 后无 unknown 终态 | crash recovery 策略确定 |
| R7 | run/event/effect/UI 可 reconcile | reconcile 脚本或测试 |

落点（Grok）：`xai-grok-shell` session / tool_calls / persistence / replay；terminal soft/hard-kill。

**当前：** 结构探测见 `scripts/verify-readiness.sh` R-struct；**完整 R0 确定性测试 = S2 未签出**。

## 3. DeepSeek readiness L0–L5

| 层 | 名称 | 通过标准 | 脚本 |
|----|------|----------|------|
| L0 | 连通 | 鉴权/路由/文本或明确 401 路由证明 | `smoke-deepseek.sh` |
| L1 | 结构化 tool call | **真实 tool_calls** + 可解析 args | `smoke-deepseek-agent.sh` |
| L2 | 最小 E2E | 同 workspace read→edit→bash | agent smoke 扩展 / 手工+证据 |
| L3 | 多轮多工具 | 连续多轮、单轮多 tool、tool error 继续 | 后续 |
| L4 | 故障与取消 | 429/断流/cancel 无重复副作用 | 后续 |
| L5 | 长会话 | compact + resume + cache 可见 | 后续 |

- `CanToolCall=true` **仅 L1 通过**  
- `ready=true` **仅 L0–L5 全过且 blockers=[]**  
- 纯文本 / HTTP 200 / 模型名可见 **不算** L1

## 4. 证据状态

`artifacts/readiness/status.json`：

| state | 含义 |
|-------|------|
| READY | 全 gate 过、blockers=[]、artifact hash 齐 |
| BLOCKED | 任一 blocker / Critical-0 失败 / 缺证据 |
| INVALIDATED | 源或 binary 漂移 |

**优先级：** 逐项结果 > aggregate > 人工总结。
