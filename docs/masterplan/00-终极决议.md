## 1. 最终战略（写死，不再辩论）

| 决策 | 选择 | 禁止 |
|------|------|------|
| 底盘 | **Fork Grok Build**（桌面 `grok-build-main`） | 不以 claw-code 为 runtime 底盘 |
| 改动哲学 | **薄壳 + 策略层 + 安全/纪律注入** | 不为优雅重写 pager/scrollback 主循环 |

| 安全 | **P0 最小硬闸 + P1 完整 Lumen 基因** | 不把安全拖到第 8 周「想起来」 |
| 任务纪律 | **Reasonix 语义**（cache / delivery / goal） | 不整仓粘贴 Reasonix Go |
| Claw | **CC_PARITY 清单 + 测试** | 不整仓合并 claw |
| 垂直 | **Go packs 独立进程/MCP** | 不为语言统一重写 Science/Oasis |
| 发布标准 | **连续自用门禁** | 不做自己不用的发布 |
| 成本 | **不计**（时间/算力/重做均可） | 不计成本 ≠ 乱加功能；加功能必须过门禁 |

## 3. 成功定义（不计成本也要的硬结果）

### 3.1 产品成功

1. 你 **连续 ≥15 个工作日** 主用 `lumen` 写真实代码，且 **想切回其他 agent ≤2 次/周**。  
2. 陌生人 **10 分钟**：安装 → 配 DeepSeek key → 改文件 → 看到工具与安全/verify 反馈。  
3. 同任务对比：体验 **不低于** 官方 grok 开源二进制；DeepSeek 长会话 **cache 可见**；危险操作 **可演示被拦**；Go 项目 **改后自测自修** 可演示。  
4. 垂直：`lumen science` / `lumen oasis`（或 MCP）**≤3 步**可达。  
5. LEGAL/NOTICE 合规；遥测默认关；无假「官方 Grok」表述。

### 3.2 工程成功

- `make check`（或等价）绿  
- `scripts/smoke-deepseek.sh` 绿  
- `scripts/smoke-security.sh` 绿  
- `policy/CC_PARITY.md` ≥80% 完成  
- eval ≥20 任务有基线数字
