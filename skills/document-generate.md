# Documentation Generation
Generate documentation for code:

1. **Read the code first**: Understand the module's public API.
2. **Package-level**: Description, key types, usage examples.
3. **Function-level**: What it does, parameters, return values, errors, examples.
4. **Follow conventions**: Match the surrounding doc style.
5. **Keep it concise**: Document the "why", not the "what" (code already shows what).

Output as a markdown document ready to be added to the project.
